## story

As a user, I want the home page to introduce the cookbook and expose the main navigation,
so that I can orient myself before exploring the routed examples.

## acceptance criteria

- [ ] When I open the application, then I see the header "Cookbook - React Router App".
- [ ] When I open the application, then the URL ends with "/" and the Home navigation item is active.
- [ ] When I open the application, then I see navigation links for "Home", "Products", "Users", "People API", and "Error Test".
- [ ] When the home page finishes loading, then I see the markdown heading "Router Cookbook".
- [ ] When I move between the top-level pages and return home, then the application header remains visible and the home content renders again without showing a route error state.
